Bot Discord Mandiri &
100% Bebas Biaya Cloud

OpenZero adalah bot Discord lokal berkinerja tinggi yang dirancang untuk berjalan langsung di PC, Home Server, atau Termux Android Anda dengan performa maksimal dan privasi penuh.

Slash Commands V2

Antarmuka modern menggunakan Discord Components V2 untuk response box yang indah dan responsif.

Privasi Penuh

Semua data konfigurasi, token, dan aktivitas chat disimpan lokal di perangkat Anda.

Tanpa Cloud API

Bebas dari limit API eksternal berbayar dan langganan hosting bulanan yang mahal.

Instalasi & Pengaturan

Ikuti langkah mudah berikut untuk menjalankan OpenZero di perangkat lokal Anda.

1

Persiapan Lingkungan

Pastikan perangkat Anda sudah terinstal Node.js (v18 ke atas) dan Anda memiliki akun Discord dengan Developer Mode aktif.

2

Klon Proyek & Instal Dependensi

Klon repositori ini ke penyimpanan lokal Anda, lalu masuk ke folder proyek dan instal dependensi.

Terminal 
git clone https://github.com/Razael-Labs/openzero-local.git
cd openzero-local
npm install
3

Konfigurasi Token Bot & Server

Duplikat file .env.example menjadi .env di root folder, kemudian isi variabel lingkungan berikut:

.env 
DISCORD_TOKEN=YOUR_BOT_TOKEN_HERE
CLIENT_ID=YOUR_BOT_APPLICATION_ID_HERE
GUILD_ID=YOUR_SERVER_GUILD_ID_HERE
Tips: Aktifkan Message Content Intent di Discord Developer Portal agar bot dapat membaca pesan untuk sistem logger lokal.
4

Jalankan Bot

Jalankan bot dalam mode pengembangan (dev) untuk auto-reload saat file diedit, atau mode produksi.

Terminal 
# Mode Development (Direkomendasikan saat dev)
npm run dev

# Mode Produksi
npm start

Manajemen Role & Template Izin

Gunakan fitur canggih /role untuk mengelola peran, mengatur hak izin, dan membuat template role khusus dengan proteksi hierarki.

Daftar Subcommand /role

/role add [user] [role]
Memberikan role ke pengguna yang dipilih. Bot akan memverifikasi tingkat hierarki agar role yang diberikan tidak melebihi posisi bot.
/role remove [user] [role]
Menghapus role dari pengguna yang dipilih dengan proteksi hierarki keamanan.
/role id [role]
Menampilkan informasi lengkap mengenai role, termasuk ID mentah, nama teks, warna hex, dan nomor posisi hierarki.
/role create [name] [template] [color]
Membuat role baru dengan memuat preset template izin (owner, admin, mods, member) atau template kustom Anda, serta warna Hex khusus.
/role settemplate [role] [template]
Reset atau ganti izin role yang sudah ada secara instan berdasarkan template tertentu.
/role savetemplate [name] [role]
Menyalin izin dari role server yang sudah ada dan menyimpannya sebagai template kustom baru ke dalam database lokal.
/role createtemplate [name] [permissions]
Membuat template kustom baru langsung dengan mendaftarkan string izin yang dipisahkan oleh koma (contoh: kick,ban,manage_messages).

4 Preset Template Default

  • owner: Memiliki hak akses Administrator penuh.
  • admin: Hak moderasi & pengaturan tingkat tinggi seperti kick, ban, kelola pesan, kelola channel, kelola server, mute, deafen, dan move member.
  • mods: Hak moderasi harian seperti kick, ban, kelola pesan, mute, deafen, dan memindahkan member.
  • member: Hak akses dasar untuk membaca channel, mengirim pesan, menyisipkan link, melampirkan file, dan berbicara di channel suara.

Cara Membuat Template Kustom

Anda dapat membuat template kustom dengan dua cara:

  1. Ekspor Role yang Ada:
    /role savetemplate name:moderator_gaming role:@Moderator
  2. Definisikan Manual:
    /role createtemplate name:helper permissions:kick,read_history,send_messages

Gunakan /role listperms untuk melihat semua string izin yang didukung bot.

Manajemen Webhook

Mudahkan proses integrasi eksternal dengan mengelola Webhook langsung dari Discord menggunakan Slash Command.

Membuat Webhook Baru

/webhook create title:[Nama Webhook] channel:[Pilih Channel] pfp:[Optional URL Foto Profil]

Membuat webhook baru secara cepat pada channel teks, pengumuman, atau suara yang ditunjuk. Fitur ini memerlukan izin ManageWebhooks.

Modern UI V2 Embed

Saat webhook berhasil dibuat, bot akan membalas dengan embed Components V2 yang menyertakan tombol interaktif "Salin URL Webhook".

Mengecek Informasi Webhook

/webhook info id_or_url:[ID atau URL Webhook]

Menganalisis dan menampilkan data webhook secara rinci berdasarkan ID webhook atau URL lengkap.

  • Nama Webhook & ID
  • Target Channel & Server
  • Pembuat Webhook (User)

Sistem Broadcast Peraturan Server

Buat tampilan peraturan server yang rapi, profesional, dan minimalis tanpa banyak emoji menggunakan skrip otomatis.

Menjalankan Skrip Aturan 
npm run send-rules

Cara Kerja & Kustomisasi Skrip:

  1. Buka file skrip pengirim aturan yang berada di src/scripts/sendRules.js.
  2. Sesuaikan isi peraturan, warna embed, dan ID channel target rules pada konstanta di dalam skrip.
  3. Jalankan skrip menggunakan perintah di atas. Skrip akan mengirimkan pesan rules yang terstruktur rapi ke server Anda menggunakan bot.
